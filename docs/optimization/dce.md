# 死代码消除（DCE）

## 算法：标记-清除

### 第一阶段：标记

从"有副作用"的指令出发，标记所有活跃指令：

```
1. 初始化：将所有指令标记为"死"
2. 从 ret, store, call, br 等有副作用指令出发
3. 递归标记其操作数的定义指令为活跃
4. 使用 Worklist 避免重复遍历
```

### 第二阶段：清除

```
1. 压缩指令数组：只保留标记为活跃的指令
2. 更新指令索引映射表
3. 更新值池中的 inst_id 引用
4. 更新基本块的指令范围
```

## 哪些指令有副作用

以下指令**不会被消除**（即使其结果未被使用）：

- `store` — 修改内存
- `call` — 可能有副作用
- `ret`, `br`, `brcond`, `switch` — 控制流
- `div`, `udiv`, `rem`, `urem` — 除零可能异常
- 所有原子指令

## 示例

```
输入:
  %1 = add const_10, const_20   ← 死代码（结果未使用）
  %2 = mul const_3, const_5     ← 死代码
  ret const_0                   ← 有副作用（控制流）

DCE 后:
  ret const_0
```

## 效果

在测试中，DCE 实现了 **61.6% 的代码缩小**（当 IR 中有大量人为冗余时）。
