# 优化 Pass 概述

Silver 的优化流水线由 6 个核心 Pass 组成，**多遍迭代直到收敛**（最多 10 遍）。

## 流水线结构

```
常量传播 → 代数简化 → 强度削弱 → 复写传播 → 死代码消除 → 窥孔优化
    ↑                                                     ↓
    └──────────────────── 迭代直到无变化 ──────────────────┘
```

## 为什么迭代

单个 Pass 的结果可能暴露新的优化机会：

```
(x + 5) + 10           ← 常量传播：无法折叠（x非常量）
    ↓
代数简化：(x + 5) + 10 → x + 15   ← 结合律
    ↓
常量传播：x + 15 → (无变化，x仍非常量)
    ↓
死代码消除：如果 x+15 未被使用 → 消除
```

只有迭代才能充分优化。

## 各 Pass 概览

| Pass | 复杂度 | 说明 |
|------|--------|------|
| 常量传播 | O(n) × 多遍 | 前向传播已知常量，折叠常量表达式 |
| 代数简化 | O(n) | 恒等式消除（x+0→x）、结合律 |
| 强度削弱 | O(n) | 昂贵操作替换（x*8→x<<3） |
| 复写传播 | O(n) × 多遍 | 多级copy链展开 |
| 死代码消除 | O(n) | 标记-清除算法 |
| 窥孔优化 | O(n) | 连续指令模式匹配 |

## 收敛性保证

每遍 Pass 都保证**指令数不会增加**，因此迭代必然终止（最多 10 遍安全上限）。
